# Introduction

Signal is a transport-agnostic application protocol for queries, mutations, and events.

Use Signal when you want:

- explicit read and write operations
- versioned names for public operations
- a standard message envelope
- retry-safe mutation handling
- duplicate-tolerant event consumption

What Signal is not:

- a general-purpose backend framework
- a workflow engine
- a transport-specific API

Start with the [envelope reference](./reference/envelope.md) and the [quickstart](./guides/quickstart.md).
